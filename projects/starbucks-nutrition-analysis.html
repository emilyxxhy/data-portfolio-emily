Starbucks Drinks Nutrition Analysis

Uncovering Ingredient-Level Insights and Developing a Healthier Drink Recommender.

#Clustering #Streamlit #KMeans #PCA

1. Project Overview & Problem

Consumers often struggle to make informed choices when faced with a large menu. The goal was to analyze nutritional data for **over 160 Starbucks beverages** using unsupervised learning techniques to group similar drinks and build a **K-Nearest Neighbors (KNN)** recommendation system that suggests healthier alternatives based on a user’s initial selection.

2. Data & Methodology

  • **Data Source:** Web scraped nutritional data (Calories, Fat, Sugar, Caffeine) for 160+ unique beverages.
  • **Data Prep:** Performed **EDA** to identify correlations between nutrients and applied **Feature Engineering** to create nutrient ratios. Data was standardized before clustering.
  • **Modeling:** Used **K-Means Clustering** ($K=5$) to segment the menu into meaningful categories. **PCA (Principal Component Analysis)** was applied to reduce dimensionality and visualize the clusters effectively.
  • **Deployment:** Integrated the Clustering and a **KNN Recommendation System** into a user-friendly **Streamlit** multi-page web application, allowing interactive data exploration.

**Actionable Insight:** The PCA visualization clearly demonstrated that the two most separated clusters were 'Black Coffee/Espresso' and 'Frappuccinos/Blended Drinks'. The KNN recommender, running in the Streamlit app, successfully suggested highly similar yet nutritionally superior alternatives (e.g., swapping a high-sugar item for a lower-sugar equivalent in the same cluster).

4. Lessons Learned

  • **PCA** was vital; without it, visualizing the differences between 10+ nutritional features would have been impossible, making the clusters difficult to interpret for the business user.
  • Deploying the analysis via **Streamlit** was key to transforming a static notebook into an interactive tool, immediately increasing the project’s user value.
