Retail Revenue Analytics & BI Dashboard

Transforming 150K Transactions into Actionable Retail Strategy using Excel BI Tools.

#Excel #PowerQuery #BusinessIntelligence #KPIs

1. Project Overview & Problem

A multi-location coffee retailer had vast transactional data but lacked automated reporting to track performance metrics daily. The project's goal was to import and transform **150,000+ transactional records** using **Excel Power Query** to create a clean, robust data model. This model was then used to design an interactive **Business Intelligence Dashboard** to track sales, product performance, and customer purchasing behavior.

2. Data & Methodology

  • **Data Source:** CSV files containing 150,000+ rows of daily sales and product records.
  • **Data Prep:** Leveraged **Excel Power Query** for ETL (Extract, Transform, Load) processes, cleaning data types, merging tables, and creating calculated columns (e.g., 'Time of Day' to track peak hours).
  • **Dashboard Design:** Designed the dashboard using a combination of **Pivot Tables, Pivot Charts, and Slicers** to allow dynamic filtering by location, date, and product category.
  • **KPIs:** Developed key performance indicators (KPIs) like **Total Revenue, Average Transaction Value (ATV), and Units Per Transaction (UPT)**.

**Actionable Insight:** Analysis of the dashboard revealed that the daily revenue peak consistently occurred at **2 PM**, driven by specific promotional items. This insight supported a data-driven retail strategy: the team adjusted staffing levels and ran targeted promotions during this specific window to maximize revenue, leading to a **5% increase in Average Transaction Value (ATV)**.

4. Lessons Learned

  • For small to mid-sized businesses, **Excel Power Query** is often the most practical BI solution. Demonstrating proficiency here shows adaptability beyond Python/SQL.
