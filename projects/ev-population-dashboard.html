Seattle EV Population Dashboard

High-Performance Geo-Spatial Analysis on 150K+ EV Records for Infrastructure Planning.

#PowerBI #BigDataViz #GeoSpatial #DAX/MQuery

1. Project Overview & Problem

The City of Seattle required actionable intelligence to support its transition to sustainable transportation, specifically regarding charging infrastructure placement and EV adoption trends. The challenge was to process a large dataset of **over 150,000 registered EV records**, clean the complex geo-spatial data, and design a high-performance, interactive dashboard (using Power BI hoáº·c Tableau) to visualize concentration, demographics, and popular models.

2. Data & Methodology

  • **Data Source:** State of Washington EV Population Data (150,000+ rows) including Make, Model, Fuel Type, and Geo-Coded Location (Zip Code).
  • **Data Prep:** Utilized **Power BI's Power Query (M Language)** for efficient data transformation, handling dirty geo-data (missing zip codes), and creating calculated columns for 'Adoption Rate'.
  • **Dashboard Focus:** Designed the dashboard around three key views: **Geo-Spatial Concentration** (Heat Map/Choropleth by Zip), **Demographics** (Adoption by County/City), and **Market Share** (Top 10 EV Models).
  • **Optimization:** Used **DAX/LOD Calculations** to ensure the large 150K dataset loaded and filtered instantly, prioritizing user experience (UX) for the City Planners.

**Actionable Insight:** The heat map instantly highlighted that **Zip Code X** had the highest EV density but the lowest public charging points per vehicle. This allowed the city planning department to prioritize the next phase of charging station deployment to this specific, underserved high-adoption area, optimizing taxpayer investment.

4. Lessons Learned

  • Visualizing **150,000+ data points** required serious **data modeling and query optimization** (DAX/MQuery). Poor performance can easily undermine the value of Big Data visualizations.
  • For public sector analysis, the **Geo-Spatial** component is non-negotiable. Connecting data points to a physical location (map) transforms data into a tangible planning tool.
